# 実装時の留意事項

## テーマと配色
- light / dark / ocean / classic の4テーマを正式サポートとする。
- テキスト色は背景と十分なコントラストを確保する。
- ページで色を直接定義するのではなく、`useToolTheme` の返却クラスを優先する。

## レイアウト
- ツールページは `ToolPageLayout` と `ToolPanel` を標準構成とする。
- 罫線や区切り線ではなく、余白と背景差で情報を区切る。
- ページ本文にサイトURLや著作権表示を重複記載しない（共通フッターに集約）。

## 検証とデプロイ
- 検証は `npm run build` を使用する。
- `npm run deploy` は本番反映のため、確認用途では実行しない。

## 例外運用
- Canvas / 画像処理 / 複雑可視化のような高度機能は例外申請を許可する。
- 例外はESLint設定内の allowlist に理由付きで登録する。
